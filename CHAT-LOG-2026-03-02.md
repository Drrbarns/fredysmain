## 5. Multi-Tenant SaaS Architecture Exploration

**User:** Hypothetically, what would we need to do to make this site a Multi-tenant Ecommerce website SaaS platform?

### Current Architecture (Single Tenant)
- One Supabase project, all tables flat with no `tenant_id`
- One Next.js app, one admin panel, hardcoded branding

### 7 Major Changes Required for Multi-Tenancy

**1. Database — Add `tenant_id` to Every Table**
- New tables: `tenants`, `tenant_users`
- Add `tenant_id` foreign key to all 30+ existing tables
- Rewrite all RLS policies to filter by `tenant_id`

**2. Middleware — Tenant Detection**
- Resolve tenant from subdomain (`store-a.yoursaas.com`) or custom domain
- Inject `x-tenant-id` header into every request
- Scope admin auth to the resolved tenant

**3. Routing — Subdomain vs Path-based**
- Subdomain routing: best UX, requires Vercel Pro (wildcard domains)
- Path-based: works on Hobby plan, less clean

**4. Dynamic Branding Per Tenant**
- Store `logo_url`, `primary_color`, `font`, `favicon` in `tenants.branding` (jsonb)
- Inject CSS variables per request
- Replace hardcoded metadata with tenant-driven values

**5. Admin Panel — Two Tiers**
- Super Admin: manage all tenants, billing, platform analytics
- Tenant Admin: existing admin panel scoped to `tenant_id`

**6. Onboarding & Billing**
- Merchant signup → subdomain → setup wizard
- Stripe Billing for subscriptions
- Plan-gated features via existing `store_modules` system

**7. All API Routes — Tenant Scoping**
- ~35 routes each need `tenant_id` filter on every query

**Effort:** ~4–6 months solo development. Database migration is the hardest part. UI transfers almost unchanged.

## 9. MCP with Self-Hosted Supabase

**User:** Does the self-hosted version give me access to MCP connection?

**Answer:** No — `mcp.supabase.com` is Supabase's cloud service and won't know about your self-hosted instance.

**Alternatives:**

**Option A — Direct PostgreSQL MCP:**
```json
{
  "mcpServers": {
    "Delizbeauty": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-postgres",
        "postgresql://postgres:PASSWORD@supabase.yourdomain.com:5432/postgres"
      ]
    }
  }
}
```

**Option B — Self-hosted Supabase MCP Server:**
```bash
npx @supabase/mcp-server-supabase \
  --supabase-url=https://supabase.yourdomain.com \
  --service-role-key=YOUR_SERVICE_ROLE_KEY
```

Both give full AI database access. Option B matches the exact cloud MCP experience.

## 10. Media Storage for SaaS

**User:** How are images and videos handled?

### Recommended Stack: Cloudflare R2

- **Zero egress fees** (vs AWS S3 at $0.09/GB out)
- Storage: $0.015/GB/month
- Free tier: 10GB + 1M requests/month
- Global CDN via Cloudflare included

**Cost for 50 tenants × 500 product images:** ~$1.87/month storage, $0 delivery.

**Folder structure per tenant:**
```
media.yourdomain.com/
  tenants/
    tenant-abc123/products/
    tenant-abc123/categories/
    tenant-abc123/branding/
    tenant-def456/products/
    ...
```

**Image optimization pipeline:**
1. Admin uploads raw photo (e.g. 4MB iPhone photo)
2. `sharp.js` resizes to 1200px max, converts to WebP, strips EXIF
3. Uploads to R2 (~120KB result)
4. Next.js `<Image>` component does final CDN delivery

**For videos:** Cloudflare Stream — $5/month base, auto-transcodes to multiple resolutions, adaptive HLS streaming.

**Total media infrastructure cost:** ~$7–10/month for entire SaaS platform regardless of tenant count.
